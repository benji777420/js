Programowanie strukturalne

Paradygmat zakładający rozdzielenie funkcjonalności programu na moduły (podprogramy), które komunikują się ze sobą poprzez określone interfejsy. Jest to rozwinięcie programowania proceduralnego poprawiające przejrzystość i jakość kodu oraz przyspieszające tworzenie oprogramowania.

Założenia

  • Bloki kodu z jednym punktem wejścia (możliwe wiele wyjść).
  • Wykonywanie wyrażeń w ustalonej kolejności.
  • Instrukcje warunkowe: if, if...else.
  • Pętle: for, while, do...while.
  • Unikanie instrukcji skoku: goto.
  • Ostrożne użycie/ograniczenie break i continue.

Programowanie obiektowe (OOP)

Paradygmat reprezentujący problemy w postaci obiektów z polami opisującymi stan i metodami definiującymi zachowanie. To podejście jest często bardziej naturalne oraz bliższe sposobowi postrzegania rzeczywistości.

Założenia

  • Abstrakcja – redukcja do kluczowych właściwości i ukrycie implementacji za interfejsem.
  • Enkapsulacja – kontrolowany dostęp do danych poprzez metody/publiczny interfejs.
  • Dziedziczenie – tworzenie nowych klas na bazie istniejących, z przejęciem i rozszerzaniem funkcji.
  • Polimorfizm – wielopostaciowość metod i interfejsów (różne implementacje, wspólny kontrakt).

Abstrakcja

Zredukowanie właściwości opisywanego obiektu do najistotniejszych i ograniczenie zakresu funkcjonalności do kluczowych działań. Implementacja jest ukryta za interfejsem – użytkownik zna „co”, a nie „jak”.

Przykład (C#)

abstract class Wielokat
{
    protected int[] boki;

    public int IleBokow()
    {
        return boki.Length;
    }

    public int Obwod()
    {
        int obwod = 0;
        foreach (int bok in boki)
        {
            obwod += bok;
        }
        return obwod;
    }

    public abstract int Pole();
}

Enkapsulacja

Dostęp do składowych jest ograniczony i odbywa się przez określony interfejs. Pola obiektów powinny być modyfikowane przez metody (gettery/settery), a nie bezpośrednio.

Dziedziczenie

Mechanizm pozwalający wywodzić nowe klasy z już istniejących, przejmując ich metody i rozszerzając lub modyfikując funkcjonalność.

Przykład (C#)

class Kwadrat : Wielokat
{
    public Kwadrat(int a)
    {
        boki = new int[4];
        boki[0] = boki[1] = boki[2] = boki[3] = a;
    }

    public override int Pole()
    {
        return boki[0] * boki[0];
    }
}

Polimorfizm

Umożliwia różne zachowania przy tym samym interfejsie/metodzie. Przykładowo przeciążenie metody ToString tak, by prawidłowo konwertowała wartości różnych typów (int, float, decimal) na ciąg znaków.

Kluczowe różnice – szybkie porównanie

Strukturalne

  • Program budowany z podprogramów (funkcji) wywoływanych w określonej kolejności.
  • Logika rozbita na moduły i procedury.
  • Silny nacisk na przepływ sterowania.

Obiektowe

  • Funkcjonalność przypisana do obiektów i ich metod.
  • Modelowanie domeny przy pomocy klas, relacji i interfejsów.
  • Ułatwia unikanie błędów jak przekazanie niewłaściwej zmiennej do funkcji (silniejszy kontekst).

Tip: W praktyce paradygmaty często się przenikają – można pisać „strukturalnie” w języku OOP i odwrotnie.
